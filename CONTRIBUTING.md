# Contributing to Govuk-Design-React

Thanks for your interest in contributing to **Govuk-Design-React**! 🚀  
Whether you’re fixing a bug, adding a new feature, improving documentation, or helping other contributors, your work is appreciated.

This guide explains **how to get started**, the workflow, and some best practices for contributing.

---

## 1. Getting Started

1. **Fork the repository** and clone it locally:

```bash
git clone https://github.com/YOUR_USERNAME/govuk-design-react.git
cd govuk-design-react
```

2. **Install dependencies**:

```bash
npm install
```

3. **Run Storybook** to see components live:

```bash
npm run storybook
```

4. **Run tests** to ensure everything works:

```bash
npm run test
```

---

## 2. How to Contribute

### A. Issues
- Check existing issues before creating a new one.  
- Look for labels like:
  - `good-first-issue` → beginner-friendly tasks  
  - `help-wanted` → tasks that need extra hands

### B. Pull Requests
1. Create a new branch for your work:

```bash
git checkout -b feature/my-new-component
```

2. Make your changes, commit, and push your branch.  
3. Open a **Pull Request (PR)** against `main`.  
4. Fill in the PR template (if available) and reference related issues.

---

## 3. Code Guidelines

- Follow the existing **React + TypeScript** patterns in the repo.  
- Use **Storybook** to document components.  
- Write tests using **Vitest** for new features or bug fixes.  
- Keep code readable, accessible, and consistent with GOV.UK design standards.

---

## 4. Documentation

- Improve existing docs or Storybook stories whenever possible.  
- Clear documentation helps new users and contributors alike.

---

## 5. Community & Support

- Ask questions or discuss ideas via **GitHub Discussions** or **issues**.  
- Be respectful and supportive — we want this to be a welcoming community for all skill levels.
